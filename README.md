# Inner Coach Bot

Личный коуч глубокой работы с собой: тёплый, честный, без сахара, с возвратом в
тело, со всеми приёмами. Не универсальный ассистент и не терапевт.

Два способа пользоваться — выбери под себя.

---

## ⭐ Способ 1 — в твоём Claude (на подписке, рекомендуем)

Есть подписка Claude (Pro/Max)? Коуч живёт прямо в приложении: без сервера, без
API, без оплаты токенов, и можно **говорить голосом**. Полная инструкция —
[USE-IN-CLAUDE.md](USE-IN-CLAUDE.md). Коротко:
- **текст, вся глубина:** создай Проект, вставь [personality.md](personality.md) в
  инструкции проекта, добавь файлы из [coach-knowledge/](coach-knowledge/) в знания;
- **голос:** вставь [voice-coach.md](voice-coach.md) в инструкции профиля и включи
  голосовой режим.

---

## Способ 2 — Telegram-бот 24/7 (на API)

Нужен коуч ботом в Telegram, который живёт на сервере и помнит между
перезапусками? Это путь на **API-ключе** (подписка для постоянного бота
ненадёжна: сессия слетает, и это против правил использования).

**Память бота:** вся переписка хранится в Postgres и переживает перезапуск; коуч
сам сохраняет важное в долгую память (паттерн, инсайт, факт) и видит её в начале
каждого нового разговора. Таблицы создаются сами. Без базы бот тоже работает, но
помнит только текущий запуск.

## Способ 2: запуск бота — позови своего агента

Программировать не нужно, всё делает Claude, вы отвечаете.

1. Нажмите **Fork** (вверху справа) — это ваша копия.
2. Склонируйте её и откройте папку в [Claude Code](https://claude.ai/code).
3. Напишите агенту: **начнём**.

Claude прочитает [CLAUDE.md](CLAUDE.md) и проведёт по шагам: бот в @BotFather →
ключ Claude → база для памяти → деплой на Railway → проверка.

```bash
git clone https://github.com/ВАШ_ЛОГИН/inner-coach-bot
cd inner-coach-bot
claude            # откроется Claude Code, дальше напишите: начнём
```

## Что понадобится

| Что | Зачем | Цена |
|-----|-------|------|
| Telegram + @BotFather | сам бот | бесплатно |
| [Anthropic API](https://console.anthropic.com) | мозг (Claude) | ~$5 на старт |
| [Railway](https://railway.app) | где бот живёт 24/7 | ~$5/мес |
| Postgres (для памяти) | чтобы помнил | бесплатно |

**База — два простых способа** (нужен только один):
- **Railway Postgres (проще):** в проекте на Railway → **New → Database → Add
  PostgreSQL**. Переменная `DATABASE_URL` подставится сама.
- **Supabase (бесплатно, портативно):** создайте проект → Settings → Database →
  Connection string (URI), вставьте в переменную `DATABASE_URL`.

---

## Переменные при деплое (Railway → Variables)

| Переменная | Что | Обязательно |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | токен из @BotFather | да |
| `ANTHROPIC_API_KEY` | ключ из console.anthropic.com | да |
| `ALLOWED_USER_IDS` | ваш Telegram id (@userinfobot) | да |
| `DATABASE_URL` | строка подключения к Postgres | для памяти |
| `COACH_PROMPT` | личность коуча целиком (приватно) | нет |
| `MODEL` / `MAX_TOKENS` / `HISTORY` | тонкая настройка | нет |

Полный список с пояснениями — в [.env.example](.env.example).

## Сделайте его своим (и не свети личное)

Личность коуча по умолчанию лежит в [personality.md](personality.md) — это тёплый
формат, готовый к работе. Хотите свой голос — отредактируйте файл, или (если форк
публичный) вставьте свою личность приватно в переменную `COACH_PROMPT`: бот берёт
её в первую очередь, и личное не уходит в репозиторий.

---

## Безопасность

- Токен, ключ и `DATABASE_URL` — **только** в Variables на Railway, не в коде.
- Обязательно заполните `ALLOWED_USER_IDS` своим id, иначе бот ответит кому угодно
  и чужие будут тратить ваш баланс Anthropic.
- В базе лежит ваша личная переписка — держите доступ к ней закрытым.
- `.env` и сессии в `.gitignore`, наружу не уходят.
- Коуч не терапевт и не врач. При кризисе он направляет к живому специалисту.
